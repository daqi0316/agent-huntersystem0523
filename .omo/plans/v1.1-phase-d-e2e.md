### 3.2 关键设计决策

| 决策 | 选项 | 选 | 理由 |
|---|---|---|---|
| HTTP 客户端 | `httpx.AsyncClient` (复用 conftest) | ✅ | 已支持 |
| MCP 工具调用 | `mcp_host.call_tool()` | ✅ | 真 MCP 协议 (v0.4e 14 server 已验) |
| Mock LLM 方式 | patch `OMLXClient` vs 传 `llm=` 参数 | ✅ 传参 | extract_from_text 已支持 `llm=` 参数, 无 patch 污染 |
| Test DB 隔离 | transaction rollback per test | ✅ 复用 conftest 的 _clear_agent_registry | DB 是真 DB (test 已用) |
| Async path 测 | 测 | ✅ 1 个 test 覆盖 | Momus §4.1 要求 |

### 3.3 Playwright Smoke 设计

```typescript
// apps/web/tests/e2e/parse-flow-smoke.spec.ts
import { test, expect } from "@playwright/test";

test("agent page renders after parse-resume API mock", async ({ page }) => {
  // 1. 注入 mock token
  await page.addInitScript(() => {
    localStorage.setItem("access_token", "mock-jwt-token");
  });
  
  // 2. 访问 /agent
  await page.goto("/agent");
  
  // 3. 验证页面渲染 (无 white screen)
  await expect(page.locator("body")).toBeVisible();
  await expect(page).toHaveURL(/\/agent/);
});
```

**约束**:
- 不依赖真后端 (用 `addInitScript` mock token, 验前端渲染)
- 不依赖真 LLM
- 1 个 test, smoke 级 (v0.4e e2e 已覆盖真后端, 这里只验前端可加载)
