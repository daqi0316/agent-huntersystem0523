# 1-2 客户白鼠 30 天启动清单

> AI 招聘助手 Phase 5 验证 — 2026-06-15 启动, 2026-07-15 收尾
> 目标: 验证 PMF, 拿到首批 5 个付费客户案例

## 1. 客户白鼠 准入门槛 (3 选 1)

候选客户必须满足:
- [ ] 真实 B2B 企业, **非** 内部测试账号
- [ ] 有活跃招聘需求 (近 30 天发布 ≥ 1 个 JD)
- [ ] 招聘 HR ≥ 1 人, 有决策权
- [ ] 愿意 30 天深入使用 + 每周 1 次 1-on-1
- [ ] 接受**数据用于案例** (脱敏后, 经书面同意)

优先级:
1. 创始人 / 朋友公司 (低决策成本)
2. 老客户 / 转介绍 (有信任)
3. 行业 KOL (有影响力)

## 2. 客户白鼠 启动会议 (D-7, 启动前 1 周)

### 2.1 会议准备
- [ ] 客户经理 1 名 (主)
- [ ] 工程师 1 名 (技术对接, 备 1h 解决问题)
- [ ] AI 评估 1 名 (协助配置评估模板)
- [ ] 30 天 onboarding 清单打印

### 2.2 会议议程 (90 分钟)
- 0-15 min: 互相介绍 + 期望对齐
- 15-30 min: 演示完整流程 (上传 JD → AI 评估 → 面试安排)
- 30-60 min: **配置客户专属** (评估模板 / 评分权重 / 通知渠道)
- 60-75 min: 试用账号开 + **首次上传 1 个真实 JD** (重点: 真实数据, 不是 demo)
- 75-90 min: Q&A + 30 天计划

### 2.3 配置清单 (客户专属)
- [ ] 组织名 + Logo
- [ ] 评估模板 (按客户行业: 互联网/零售/金融 3 套默认)
- [ ] 评分权重 (客户可调, 默认 4 维度均等)
- [ ] 通知渠道 (微信/短信/in-app 至少 1 个)
- [ ] 面试官账号 (一般 2-5 个, 角色: Interviewer)
- [ ] 健康度规则 (按客户调阈值)

## 3. 30 天 onboarding 节奏 (D1 / D3 / D7 / D14 / D21 / D30)

### D1 — 启动日
- [ ] 客户完成登录 + 接受法务 (P5-9)
- [ ] 上传 1 个真实 JD (我们协助)
- [ ] 试用 14 天开始计时 (P6-3)
- [ ] 推站内信 D+1 (P6-5)
- [ ] 客户白鼠专属飞书群建立 (含客服 + 工程师)

### D3 — 第 1 次 1-on-1
- [ ] 15 min 视频: 检查是否完成 onboarding
- [ ] 引导上传更多 JD (目标: ≥ 3 个)
- [ ] 推 D+3 站内信 (P6-5)
- [ ] 答疑

### D7 — 中期检查
- [ ] 视频: 健康度评分是否 ≥ 50
- [ ] 引导首次 AI 评估 (目标: ≥ 5 个候选人评估)
- [ ] 推 D+7 站内信 + 短信 (试用剩 7 天, P6-5)
- [ ] 收集客户反馈 (问卷 5 题)

### D11 — 试用到期预警
- [ ] 推 D+11 试用到期前 3 天站内信 + 短信 (P6-5)
- [ ] 视频: 续订讨论 (按健康度 + 反馈)
- [ ] 续订方案: 8 折 + 1 个月高级版 (P6-4 老带新)

### D14 — 试用到期
- [ ] 推 D+14 站内信 + 短信 + 微信模板 (P6-5)
- [ ] 试用降级 (只读 + 数据保留 30 天)
- [ ] 客户决策: 续订 / 暂停 / 流失
- [ ] 案例采集 (经同意) → P6-10 案例研究页

### D21 — 长期健康度
- [ ] 视频: 周活 / 健康度 / 续费意愿
- [ ] 引导团队协作 (邀请同事)
- [ ] P6-4 老带新推荐码 (双向 seat+1)

### D30 — 收尾
- [ ] 视频: 30 天回顾 + 续费决策
- [ ] 收集: NPS / 推荐意愿 / 案例素材
- [ ] 决策:
  - 续费 → 转付费客户 (P6-3)
  - 暂停 → 健康度 < 50, 调研流失原因
  - 流失 → 30 天后删数据, 记录原因

## 5. 风险预案 (出问题怎么办)

### 5.1 客户卡在 D7
**信号**: 健康度 < 30, WAU < 20%
**行动**:
- 24h 内 1-on-1 紧急会议
- 工程 + AI 经理上门协助
- 必要时换 onboarding 客户经理

### 5.2 AI 评估被投诉
**信号**: 客户或候选人对 AI 评分有异议
**行动**:
- 24h 内人工复核 (P5-10)
- 误判案例入训练黑名单
- 给客户书面结论 + 道歉

### 5.3 客户想流失
**信号**: D14 健康度 < 30, 续费意愿 < 5
**行动**:
- 1-on-1 调研流失原因
- 询问: 价格 / 功能 / 体验 / 竞品?
- 提供: 折扣 / 定制 / 暂停 30 天
- 不强留, 保留关系 (未来回流)

## 6. 1-2 客户白鼠 候选人画像

### 理想候选 1: 中型互联网公司
- 规模: 100-500 人
- HR 团队: 3-8 人
- 月招聘: 50-200 个
- 痛点: 工程岗位简历筛不动, AI 评估需求强
- 决策人: HR 总监 / CTO

### 理想候选 2: 连锁零售 / 服务业
- 规模: 1000+ 人, 多门店
- HR 团队: 5-15 人
- 月招聘: 100+ 门店岗位
- 痛点: 店长 / 督导标准化招聘, 区域协作
- 决策人: 招聘总监 / HRVP

### 理想候选 3: 金融 / 财富管理
- 规模: 100-300 人
- HR 团队: 2-5 人
- 月招聘: 30-50 个
- 痛点: 合规 + AI 评估透明度
- 决策人: 合规总监 / HRD

## 10. 失败兜底 (如果 1-2 客户白鼠失败)

- D14 健康度 < 30 → 立即换第 2 候选
- D30 流失率 > 50% → 暂停扩招, 优化产品
- 资金链紧张 → 启动种子轮 (目标 6-8M)
- 产品重大缺陷 → 暂停客户白鼠, 6 周迭代后重启
