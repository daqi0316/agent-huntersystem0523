# 退款 / 争议处理 SOP (P6-11)

更新时间: 2026-06-06
适用: 微信支付 + 支付宝 dispute + 客服主动退款

## 1. 触发场景

| 场景 | SLA | 责任 |
|---|---|---|
| 客户主动申请退款 (试用不满意 / 多付 / 重复扣款) | 24h 内处理 | 客户成功 |
| 微信支付 dispute (用户在微信内点 "投诉") | 7d 内响应, 24h 内响应最佳 | 客户成功 + 财务 |
| 支付宝 dispute (用户在支付宝内点 "投诉") | 7d 内响应 | 客户成功 + 财务 |
| 信用卡 chargeback (海外, 未来场景) | 30d 响应 | 财务 |

## 2. 主动退款流程 (客户申请)

### 2.1 客户成功接收
1. 客户微信 / 飞书 / 站内消息联系
2. 收集: 客户 ID / 订单 out_trade_no / 退款原因
3. 录入工单系统 (美洽/智齿, P6-6 ship 后)

### 2.2 资格判断 (7d 窗口)
- P5-3 退款 API: 仅 7d 内可退
- 超 7d: 走客服协商, 可走部分退款 + 保留服务

### 2.3 退款 API 调用
```bash
curl -X POST https://api.airecruit.com/api/v1/payment/orders/<out_trade_no>/refund \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "refund_amount_cents": 29900,
    "reason": "客户申请 7d 试用不满意"
  }'
```

### 2.4 退款后
- audit_log 落 PAYMENT_REFUND
- subscription 标 CANCELLED (若是当期最后一笔)
- 客户成功飞书消息客户: "已退 ¥299, 1-3 工作日到账"

## 3. 微信支付 dispute 处理

### 3.1 接收通知
微信商户平台 → 通知中心 → "消费者投诉"
或 微信支付 API 推送: `complaints_api`

### 3.2 7d 内响应
登录商户平台 → 投诉详情:
- 查看投诉原因 (一般是 "未提供服务" / "服务不满意" / "重复扣款")
- 上传证据:
  - 服务使用记录 (audit_log 截图)
  - 服务协议截图
  - 客户确认收到服务证据 (微信 / 站内消息)
- 提交: "同意退款" 或 "拒绝投诉" + 证据

### 3.3 决策矩阵
| 投诉原因 | 证据 | 决策 |
|---|---|---|
| 未提供服务 | audit_log 完整 | 拒绝投诉 + 证据 |
| 服务不满意 | 客户已确认收货 | 部分退款 (协商 30-70%) |
| 重复扣款 | 实际只 1 笔 | 同意退款 (全额) |
| 未授权扣款 | 客户主动续费 | 同意退款 (全额) |

## 4. 支付宝 dispute 处理

### 4.1 接收通知
支付宝商家中心 → 客户中心 → 投诉管理
或 API: `alipay.trade.order.settle`

### 4.2 流程
- 72h 内响应 (支付宝 SLA 严于微信)
- 上传证据 (服务协议 + 使用记录)
- 决策同微信 (3.3 决策矩阵)

## 5. 退款 / 争议 监控

### 5.1 飞书告警 (P5-7 集成)
- 退款 > ¥1000 单笔 → 飞书 P1 财务群
- 7d 内退款 > 3 笔同 org → 飞书 P1 客户成功
- 月退款率 > 10% → 飞书 P2 财务 + PM 群

### 5.2 月度对账
- 每月 1 号 02:00 自动跑 scripts/backup-postgres.sh + 月度账单
- 客户成功团队 review 退款 case (避免重复问题)

## 6. 退款接口 (P5-3 已 ship)

| 端点 | 方法 | 说明 |
|---|---|---|
| `/api/v1/payment/orders/<out_trade_no>/refund` | POST | 主动退款 |
| `/api/v1/payment/orders/<out_trade_no>` | GET | 查订单状态 |
| `/api/v1/subscription/change-plan` | POST | 升降级 (产生新订单/退款) |

## 7. 边界情况

| 情况 | 处理 |
|---|---|
| 客户已用满月服务再退 | 部分退款 (按未用天数) |
| 微信退款超过原路 | 退到零钱 (默认), 需客户重新提供银行卡 |
| 客户反复 dispute (恶意) | 黑名单 + 拒绝服务 |
| 退款时客户已删账号 | 退到原支付账户 (PII 保留) |
| 跨年订单退款 | 按原始金额 + 财务做账 |

## 8. 升级路径

- 退款 > ¥5000: 财务审批
- 退款 > ¥10000: PM 审批
- 客户持续 dispute: 法务介入 (P5-9 律师)
- 商户号冻结: 启动应急预案 (走银行对公转账 7d 内)
