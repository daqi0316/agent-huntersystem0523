# AI Recruitment System — 2026 H2 ~ 2027 H1 路线图 (国内版 v2)

> 6 个月路线图:从"工程跑通"到"商业化上线 + 增长飞轮"。
> 时间窗:2026-07 ~ 2027-01。
> **约束**:国内 only / 无邮件营销 / 无对公转账 / **纯 SaaS 多租户 (本地化部署推 Phase 6+ 客户驱动)**。
> 假设:1 个 full-stack (我) + 1 个产品/运营 (你 or 招聘)。超出此规模需扩人。

## 0. 当前状态盘点 (2026-06 末)

| 维度 | 状态 | 说明 |
|---|---|---|
| **核心产品** | 70% | 候选人/职位/申请/简历筛选/评估/面试/看板 全部跑通,但**无多租户** |
| **AI 能力** | 80% | 简历解析/智能匹配/多维评估/Orchestrator/Human-in-Loop 可用,准确率待产品化打磨 |
| **基础设施** | 60% | Docker compose + CI 有,无 Dockerfile + staging/prod 分层,监控有埋点但无告警 |
| **商用化** | 5% | 单租户 + 单用户,无 team/org,无支付,无合规 |
| **文档** | 80% | README + 4 份 plan + 4 份工程文档 + lessons-learned |
| **测试** | 75% | 后端 pytest 10/10 agent_events,前端 25 单测 + 26 E2E,但**无 cross-package 集成 e2e** |
| **MCP v4 架构** | 90% | **PR-8 ship (2026-06-06)** — dual-track supervisor pilot 2 工具跑通,5/5 性能预算,4/4 故障注入,git tag mcp-v4-pr8-pre。PR-9 38 工具 scale 待启动 (9.5d) |

**关键风险**:不上 Phase 4 → 产品永远 demo 阶段,签不下 B2B 合同 (无多租户 = 数据隔离隐患)。

## 1. Phase 5 (月 1-2): 商用化基础 (硬阻塞)

**目标**:能接第一个付费 B2B 客户 (5-50 人 HR 团队)。
**北极星**:**3-5 个付费客户签约** + 1 个月稳定使用无 P0 事故。

### 1.1 关键任务

| ID | 任务 | 工时 | 验收 |
|---|---|---|---|
| P5-1 | **多租户模型** — Organization / Membership / RLS + 跨租户测试 | **12d** | 25+ 表加 `org_id`,RLS policy,跨租户 negative test 100% |
| P5-2 | **Team 管理** — 邀请/角色 (Owner/HR/Viewer) + 微信扫码登录 | 3d | 邀请/角色生效,微信 OAuth 跑通 |
| P5-3 | **国内支付** — 微信支付 (商户号) + 支付宝 (商户号) | **6d** | 沙箱跑通订阅/续费/降级,前端 paywall |
| P5-4 | **个保法合规** — 数据导出 (Art. 15) + 删除 (Art. 17) + Cookie/隐私政策 | **12d** | 导出关联 20+ 表,删除 30 天宽限,律师审 |
| P5-5 | **审计日志** — 用户/管理员操作全留痕 (who/what/when/ip) | 2d | audit_log 表 + admin UI + 6 个月保留 |
| P5-6 | **生产部署** — Dockerfile + 阿里云 ACK + staging + 域名/SSL + ICP 备案 | 6d | staging.airecruit.com 可访问,prod SOP |
| P5-7 | **监控告警** — /metrics + 告警规则 + Sentry + 飞书/钉钉 webhook | 2d | error rate / latency / 5xx 告警飞书群 |
| P5-8 | **配額/限流** — per-org quota (API/storage/users/LLM token) | 2d | 超过自动 429 + 飞书通知 owner |
| P5-9 | **法务文件** — 服务条款 / 隐私政策 / DPA 模板 (国内律师) | 4d | 律师出稿 + 文档站 + signup 同意 |
| P5-10 | **AI 监管合规** — 2026-08 生成式 AI 服务管理办法 (内容标识/人工审核/可申诉) | 3d | 候选人 AI 评分有来源标识,人工可覆盖,可申诉 |
| P5-11 | **反垃圾/反滥用** — 短信验证 + 算力限流 + 邀请奖励防刷 | 2d | 1 手机号 1 账号,LLM token 超限熔断 |
| P5-12 | **老数据迁移** — 单租户 → 多租户 default org 自动建 | 3d | 自动建 default org,邀请现有用户加入 |
| P5-13 | **回滚方案** — 部署/staging 出问题 5 分钟回滚 | 1d | 蓝绿/灰度 + DB migration 可逆 |
| P5-14 | **ICP 备案** — 域名备案 (微信支付/小程序前置) | 1d 申请 + 1-2 周等待 | 备案号到位 |
| P5-15 | **第一个客户 onboarding runbook** | 1d | 30 天 onboarding 清单 + KPI |

**合计**:58d ≈ **12 周 (含 buffer 实际 9-10 周)**。Momus 校正后真实工期。

### 1.2 团队需求

- 你 (PM/运营) → 客户沟通 + 微信支付/支付宝商户号申请 + 律师对接 + ICP 备案

## 2. Phase 6 (月 3-4): 增长 + 转化

**目标**:从 1-5 客户 → 10 客户,跑通获客/转化/留存闭环。
**北极星**:MRR ¥15K (10 客户 avg ¥1.5K/月) + 周活 60%。
**获客渠道**:百度 SEO + 知乎/公众号内容 + 钉钉/飞书 HR 群运营 + 老客户介绍。
**触达方式 (无邮件)**:站内信 + 微信模板消息 (服务号) + 短信 (重要通知)。

| ID | 任务 | 工时 | 验收 |
|---|---|---|---|

| P6-2 | **Onboarding 流** — 注册 → 引导 → 第一个候选人 5 分钟内 (无邮件) | 3d | 站内信 + 微信模板引导,完成率 > 70% |
| P6-3 | **Self-serve signup** — 手机号 + 微信扫码 + 14 天试用 | 2d | 无人工介入即可试用 |
| P6-4 | **产品内 growth loop** — 邀请同事奖励 (seat +1) | 2d | 邀请转化率 > 20% |
| P6-5 | **触达自动化 (无邮件)** — 站内信 + 微信模板消息 + 短信 (D+1/D+3/D+7/D+14) | 4d | 微信服务号 + 短信服务商 (阿里云/腾讯云) |

**合计**:45d ≈ **9 周**。

### 2.2 关键风险 + 缓解
| 风险 | 影响 | 缓解 |
|---|---|---|
| 自助注册后无引导 → 流失 | 留存 | P6-2 引导流 + P6-5 微信/短信触达 |

## 5. 跨阶段关键事项

### 5.1 资金需求
- Phase 5-6: 自有资金 + 朋友借款 (¥100K 内) — 撑到 10 客户
- Phase 7: 种子轮 ¥1-3M (团队扩展 + 多 region + AI 研发)
- Phase 8: A 轮 ¥10-30M (Enterprise 销售 + 等保 + 信创适配)

### 5.2 招聘优先级
- Phase 5-6: 不招
- Phase 7 月 5: AI 工程师
- Phase 8: 销售 + CS + 设计 + DevOps

### 5.3 技术债 (需在 Phase 5 同时偿还)
- 单租户 → 多租户 改造
- localStorage → 远端持久化
- 单元测试覆盖率 75% → 85%
- API 文档 (OpenAPI → Swagger UI 国内 CDN)
- 海外 LLM API (OpenAI) → 全部迁国内 (智谱/通义千问/文心一言)

### 5.4 不做清单 (本期)
- ❌ 海外市场 / 出海 / 国际版
- ❌ 邮件营销 (站内信 + 微信模板 + 短信 替代)
- ❌ 对公转账支付 (微信 + 支付宝 够 Phase 5-7,Phase 8 Enterprise 再说)
- ❌ 移动原生 App (PWA 够用)
- ❌ 视频面试 (WebRTC 重投入,ROI 低)
- ❌ 简历代写 (合规风险)
- ❌ "AI 自动 offer" (合规风险,人必须最终决策)
- ❌ "候选人评分公开" (反歧视 + 隐私,只能给雇主内部用)

### 5.5 私有化部署 — 条件触发 (Phase 6+)

**默认不做 Phase 5**。**触发条件** (满足任一):
- 累计 3+ 客户主动询问本地化部署
- 1 个 enterprise 客户 (¥100K+/年) 因"数据必须内网"卡住谈判
- 法规要求 (等保三级实施后, 某些行业强制)

**最小化交付 (1 周)**:
- `docker-compose.local.yml` (一键起, 关 RLS, 单租户模式)
- license JSON 文件 (启动校验, 过期不锁, 仅警告)
- 部署手册 + 录屏

**完整版 (3 周)** — Phase 7 后, 真有 enterprise 客户时:
- RSA 签名 license + 离线心跳
- 升级 SOP + 灰度发布
- 1 周运维培训

**为什么放条件触发**:
- 当前 0 客户, 不知道 enterprise vs SMB 比例
- SMB SaaS 决策快 (1 周), enterprise 决策慢 (3-6 月)
- 工程 ROI: 6 月内 SaaS 拿 1-3 付费 > 本地化拿 0 POC
- "1 周最小版" 留作应急: 客户问起, 1 周能补

**架构原则 (避免 anti-pattern)**:
- 同一 codebase, 不引入 `DEPLOYMENT_MODE` 开关
- SaaS 部署启用 RLS + 多租户 migrations;本地化部署**跳过**多租户 migrations
- 业务 endpoint 内部 `if SINGLE_TENANT: 跳过 get_org_context`
- Alembic migration 条件化 + 启动时配置, 不散落 if/else

**反例 (不要做的事)**:
- ❌ 不要"1 codebase 2 modes" 散落 if/else (测试矩阵 2x)
- ❌ 不要提前做 license 加密 (Phase 5 简单版够)
- ❌ 不要给客户"半年免费升级"承诺 (semver 锁版本)
