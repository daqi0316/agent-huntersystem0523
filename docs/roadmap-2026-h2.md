# AI Recruitment System — 2026 H2 ~ 2027 H1 路线图

> 6 个月路线图:从"工程跑通"到"商业化上线 + 增长飞轮"。
> 时间窗:2026-07 ~ 2027-01。
> 假设:1 个 full-stack (我) + 1 个产品/运营 (你 or 招聘)。超出此规模需扩人。

## 0. 当前状态盘点 (2026-06 末)

| 维度 | 状态 | 说明 |
|---|---|---|
| **核心产品** | 70% | 候选人/职位/申请/简历筛选/评估/面试/看板 全部跑通,但**无多租户** |
| **AI 能力** | 80% | 简历解析/智能匹配/多维评估/Orchestrator/Human-in-Loop 可用,准确率待产品化打磨 |
| **基础设施** | 60% | Docker compose + CI 有,无 Dockerfile + staging/prod 分层,监控有埋点 (T6) 但无告警 |
| **商用化** | 5% | 单租户 + 单用户,无 team/org,无 Stripe,无 GDPR 流程 |
| **文档** | 80% | README + 4 份 plan + 4 份工程文档 + lessons-learned |
| **测试** | 75% | 后端 pytest 10/10 agent_events,前端 25 单测 + 26 E2E,但**无 cross-package 集成 e2e** |

**关键风险**:不上 Phase 4 → 产品永远 demo 阶段,签不下 B2B 合同 (无多租户 = 数据隔离隐患)。

## 1. Phase 5 (月 1-2): 商用化基础 (硬阻塞)

**目标**:能接第一个付费 B2B 客户 (5-50 人 HR 团队)。
**北极星**:1 个付费客户签约 + 1 个月稳定使用无 P0 事故。

### 1.1 关键任务

| ID | 任务 | 工时 | 验收 |
|---|---|---|---|
| P5-1 | **多租户模型** — Organization / Membership / Org-scoped queries | 5d | 所有 model 加 `org_id`,RLS policy,迁移脚本,跨租户测试 |
| P5-2 | **Team 管理** — 邀请/角色 (Owner/HR/Viewer) / SSO prep | 3d | UI + API + 邀请邮件 (Resend) |
| P5-3 | **Stripe 集成** — Plan (Starter/Pro/Enterprise) + 订阅 + Webhook | 4d | 沙箱跑通订阅/续费/降级/取消,前端 paywall |
| P5-4 | **GDPR 合规** — 数据导出 (Art. 15) + 删除 (Art. 17) + Cookie consent | 3d | 用户可导出自己数据,删除账号 30 天宽限 |
| P5-5 | **审计日志** — 用户/管理员操作全留痕 (who/what/when/ip) | 2d | audit_log 表 + admin 查询 UI + 6 个月保留 |
| P5-6 | **生产部署** — Dockerfile + fly.io/Render 部署 + staging env + 域名/SSL | 4d | staging.airecruit.com 可访问,prod 部署 SOP |
| P5-7 | **监控告警** — /metrics (T6 已做) + 告警规则 + 错误追踪 (Sentry) | 2d | error rate / latency / 5xx 告警飞书/邮件 |
| P5-8 | **配額/限流** — per-org quota (API call / storage / users) | 2d | 超过自动 429 + 邮件通知 owner |
| P5-9 | **法务/合规文件** — 服务条款 / 隐私政策 / DPA 模板 | 2d | 律师审 + 文档站 + signup 同意 |
| P5-10 | **第一个客户 onboarding runbook** | 1d | 30 天 onboarding 清单 + 客户成功 KPI |

**合计**:28 天 ≈ 6 周,含 buffer 8 周。

### 1.2 团队需求
- 我 (full-stack) → 全部实现
- 你 (PM/运营) → 客户沟通 + Stripe 申请 + 法务协调

### 1.3 关键风险 + 缓解
| 风险 | 影响 | 缓解 |
|---|---|---|
| 多租户 RLS 漏 → 数据泄漏 | 致命 | 跨租户测试 + 第三方安全审计 (P5-1 必带) |
| Stripe 申请被拒 (国内主体) | 阻塞签约 | 备选:Creem/Lemon Squeezy (海外)/微信支付 (国内) |
| GDPR 罚款 | 法律 | 律师审 + 默认不收集 PII (T6 已铺垫) |
| 第一个客户决策链长 (法务/IT 采购) | 时间 | 提供 self-serve 试用 14 天 + 模板合同 |

### 1.4 OKR

**Objective**: 商用化基础设施 ready,可承接 B2B 客户
- KR1: 多租户 + Stripe + GDPR 全部上线 (10/10 验收 checklist)
- KR2: staging + prod 双环境跑通,健康检查 < 5 分钟定位故障
- KR3: 1 个付费客户签约 (哪怕 ¥1 试用)
- KR4: 监控覆盖 100% P0 路径,告警 P0 → 5 分钟内 oncall 知道

## 2. Phase 6 (月 3-4): 增长 + 转化

**目标**:从 1 个客户 → 10 个客户,跑通获客/转化/留存闭环。
**北极星**:MRR ¥10K (10 个客户 × ¥1K/月) + 周活 60%。

### 2.1 关键任务

| ID | 任务 | 工时 | 验收 |
|---|---|---|---|
| P6-1 | **Marketing site** — Next.js 静态站,SEO 优化,博客 | 5d | 首页/功能页/定价页/案例 + 5 篇 SEO 博客 |
| P6-2 | **Onboarding 流** — 注册 → 引导 → 第一个候选人 5 分钟内 | 3d | 漏斗指标埋点,完成率 > 70% |
| P6-3 | **Self-serve signup** — email + OAuth (Google) + 14 天试用 | 2d | 无需人工介入即可试用 |
| P6-4 | **产品内 growth loop** — 邀请同事奖励 (seat +1) | 2d | 邀请转化率 > 20% |
| P6-5 | **邮件自动化** — 试用 D+1/D+3/D+7/D+14 触达 + 续费提醒 | 3d | Resend + 模板 + 退订 |
| P6-6 | **客户支持工具** — 站内 chat (Crisp) + 知识库 (Notion) + 工单 | 2d | 首响 < 4h |
| P6-7 | **A/B 测试框架** — 定价页/CTA 文案/注册流 | 3d | 至少 1 个 A/B 上线,有结论 |
| P6-8 | **集成市场 MVP** — 简历解析/邮件/日历/钉钉/飞书 | 5d | 至少 3 个集成,OAuth 接通 |
| P6-9 | **数据看板 (内部)** — CAC / LTV / Churn / NPS 周报 | 2d | 自动周报邮件给团队 |
| P6-10 | **案例研究** — 第一个客户的 before/after 数据 + 视频 | 2d | 1 个深度 case,可放官网 |

**合计**:29 天 ≈ 6 周,含 buffer 8 周。

### 2.2 关键风险 + 缓解
| 风险 | 影响 | 缓解 |
|---|---|---|
| 自助注册后无引导 → 流失 | 留存 | P6-2 引导流 + P6-5 邮件触达 |
| 集成开发量低估 | 时间 | 优先钉钉/飞书 (国内 HR 高频) |
| 营销内容产出慢 | SEO 滞后 | AI 辅助 + 1 篇/周最低 |
| 客户成功失败 → 第一个客户流失 | 口碑 | P5-10 runbook + 主动 check-in |

### 2.3 OKR
- KR1: 10 个付费客户 (含 1 个 ¥10K/月 enterprise)
- KR2: 周活 > 60% (DAU/WAU)
- KR3: NPS > 30 (首批客户调研)
- KR4: 获客成本 CAC < ¥500 (SEO + 内容为主)

## 3. Phase 7 (月 5-6): 规模 + 深度

**目标**:从 10 客户 → 30 客户,产品深度提升 (AI 准确率 + 行业化)。
**北极星**:MRR ¥50K + NRR > 120% (扩张 > 流失)。

### 3.1 关键任务

| ID | 任务 | 工时 | 验收 |
|---|---|---|---|
| P7-1 | **AI 准确率提升** — 简历解析 F1 > 95%,评估相关性 > 0.8 | 8d | golden set 自动化评估 + 持续回归 |
| P7-2 | **行业化模板** — 互联网/金融/制造/教育 各 1 个 JD 模板 | 4d | 4 个行业模板 + 行业评估维度 |
| P7-3 | **多语言** — 中英双语 UI + 简历解析 | 5d | zh/en 切换,英文简历解析 F1 > 90% |
| P7-4 | **API for partners** — 公开 API + 文档 + 限流 | 4d | 5 个 partner 接通 |
| P7-5 | **Webhook + 事件订阅** — 客户系统集成 | 3d | 至少 5 种事件,重试 + 死信 |
| P7-6 | **高级评估** — 技能图谱 + 能力雷达图 | 5d | 候选人能力可视化 |
| P7-7 | **面试 AI 助手** — 实时问题推荐 + 评分 (Human-in-Loop) | 6d | 面试官反馈 > 4/5 |
| P7-8 | **Mobile PWA** — 移动端读消息/审批 (不做原生) | 4d | PWA 安装 < 5s,核心场景可用 |
| P7-9 | **多区域部署** — 国内/海外双 region (国内阿里云 + 海外 fly.io) | 5d | 跨 region 延迟 < 200ms |
| P7-10 | **SOC2 Type I 准备** — 安全审计 + 文档 + 流程 | 8d (并行) | 至少 1 家审计启动 |

**合计**:47 天 ≈ 9-10 周,需并行 2 人。

### 3.2 团队扩展
- 月 5: 加 1 个 AI 工程师 (评估准确率)
- 月 6: 加 1 个 DevOps (多区域 + SOC2)

### 3.3 OKR
- KR1: 30 个客户,MRR ¥50K
- KR2: NRR > 120% (扩张收入 > 流失)
- KR3: 简历解析 F1 > 95%
- KR4: 集成 partner ≥ 5

## 4. Phase 8 (月 7-12): Enterprise + 护城河

**目标**:签下 1-2 个 enterprise 客户 (100-1000 人),建立护城河。
**北极星**:1 个 enterprise 客户 (¥100K+/年) + SOC2 Type I 拿到。

### 4.1 关键任务 (示意,详细 plan 月 7 再定)
- **Enterprise 功能**:SAML SSO / SCIM / 审计日志导出 / 自定义品牌 / 私有部署
- **SOC2 Type I** 拿到
- **行业纵深**:1 个行业 (如互联网技术招聘) 做到 top-of-mind
- **数据护城河**:行业人才库 (脱敏后) + 趋势报告
- **生态**:招聘公司代理 / 猎头平台合作

### 4.2 团队规模
- 5-8 人:full-stack × 2, AI × 1, DevOps × 1, 设计 × 1, 销售 × 1, CS × 1

## 5. 跨阶段关键事项

### 5.1 资金需求
- Phase 5-6: 自有资金 + 朋友借款 (¥100K 内) — 撑到 10 客户
- Phase 7: 种子轮 ¥1-3M (团队扩展 + 多区域 + AI 研发)
- Phase 8: A 轮 ¥10-30M (Enterprise 销售 + SOC2 + 行业纵深)

### 5.2 招聘优先级
- Phase 5: 不招
- Phase 6: 不招
- Phase 7 月 5: AI 工程师 + DevOps
- Phase 8: 销售 + CS + 设计

### 5.3 技术债 (需在 Phase 5 同时偿还)
- 单租户 → 多租户 改造
- localStorage → 远端持久化
- 单元测试覆盖率 75% → 85%
- API 文档 (OpenAPI → Swagger UI)

### 5.4 不做清单 (本期)
- ❌ 移动原生 App (PWA 够用)
- ❌ 视频面试 (WebRTC 重投入,ROI 低)
- ❌ 简历代写 (合规风险)
- ❌ 海外市场拓展 (Phase 8+ 再看)
- ❌ 区块链/Web3 方向

## 10. 一句话总结

> 商用化 = **多租户 (技术) + 计费 (商业) + 合规 (法律) + 部署 (运维)**,缺一不可。
> 6 个月节奏:Month 1-2 上商用化 → Month 3-4 拉新客 → Month 5-6 做深 + 扩规模。
> 假设 1-2 人小团队 + ¥100K 内启动资金,12 个月可达 MRR ¥50K + 30 客户 + SOC2 准备。
