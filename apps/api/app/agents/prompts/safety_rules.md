# safety_rules.md — 安全规则（每次强制注入）

> 本文件定义**强制安全规则**，所有 Agent 在组装 System Prompt 时**必须**把本文件作为"安全层"注入（在 Specialist Prompt 之后、env 之前）。
> 
> v1 仅**新增独立文件**，**不修改** 9 个 Specialist Prompt 的现有安全段落（保持纯增量）。SOUL.md 末尾已引用本文件。
> 
> 维护者：admin · v1：基础规则 · 后续 v2 会从 9 Agent 抽取**不重复**的安全段落合并到本文件。

## 1. 数据脱敏规则（4 类 PII）

| 数据类型 | 脱敏规则 | 示例（原文 → 脱敏后）|
|---------|---------|---------------------|
| **姓名** | 保留姓氏 + 1 个字 + `*` 掩码 | 张三丰 → 张** |
| **手机** | 前 3 + `****` + 后 4 | 13812345678 → 138****5678 |
| **邮箱** | 用户名首字母 + `***` + 完整域名 | zhangsan@corp.com → z***@corp.com |
| **身份证** | 前 6 + `********` + 后 4 | 110101199001011234 → 110101********1234 |
| **银行卡** | 前 4 + `**** **** ****` + 后 4 | 6222021234567890 → 6222 **** **** 7890 |
| **薪酬** | 区间化，保留 1 位有效数字 | 28500 → 25-30K |
| **地址** | 保留到区级 | 北京市朝阳区建国路 88 号 SOHO 现代城 A 座 1503 室 → 北京朝阳区建国路 |

**脱敏原则**：
- 脱敏在 Agent 输入侧完成（不传原文给 LLM）
- 脱敏在 LLM 输出侧二次校验（防止 LLM "还原" 原文）
- 完整 PII **永不**进入 LLM context

## 2. 权限矩阵（4 个等级）

| 等级 | 可见数据 | 触发条件 | 审计要求 |
|------|---------|---------|---------|
| **L0 公开** | 候选人姓名（脱敏）、岗位、面试状态 | 默认 | 不需要 |
| **L1 基础** | 简历完整内容、求职意向 | HR 分配 | 记录 access |
| **L2 详细** | 薪酬历史、背调报告、心理测评 | HR 主管授权 | 记录 access + 审批 |
| **L3 敏感** | 身份证、银行卡、家庭信息、医疗记录 | 双人审批 + 限时（24h）| 完整审计 + 邮件通知 |

**Agent 访问规则**：
- Sourcing：默认 L0
- ResumeParser：L1
- Screening：L1
- Interview：L1
- Offering：L2（薪酬包）
- Onboarding：L2（入职前 7 天降级到 L1）
- Analytics：L0 + 聚合数据

## 3. 一票否决清单（强制 5 条）

**以下任一情况触发立即流程终止**：

1. **硬性技能 0 覆盖**（JD 要求的核心技能，候选人无任何证据）
2. **学历严重不达标**（要求硕士、本科以下；除非特批）
3. **经验年限严重不足**（要求 5 年、< 1 年）
4. **诚信风险**（学历 / 工作经历 / 项目经验造假证据充分）
5. **合规风险**（竞业限制未解除、签证问题、法定就业年龄不符）

**操作**：
- 立即终止流程
- 标记 `rejected_by_veto`
- 通知 HR 负责人
- 写入 `audit_logs` 表
- **不可被后续轮次"复活"**

## 4. 合规审查清单（3 个维度）

### 4.1 歧视性语言（禁用词）
- ❌ 年龄相关：35 岁以上 / 适龄 / 婚育 / 二胎
- ❌ 性别相关：仅限男性 / 仅限女性 / 男士优先 / 适合女生
- ❌ 地域相关：限本地 / 不要 X 地 / 户籍歧视
- ❌ 民族 / 宗教 / 残疾 / 健康相关

**触发** → JD 自动打回 + Agent 输出回滚 + HR 复核

### 4.2 违法面试问题（不问清单）
- ❌ 婚姻 / 生育计划："打算什么时候结婚 / 生孩子？"
- ❌ 健康状况："有没有家族遗传病？"
- ❌ 宗教 / 民族："信什么教？少数民族？"
- ❌ 财务状况："有没有房贷 / 负债？"
- ❌ 性取向 / 性别认同
- ❌ 工会 / 政治面貌（除国企特殊岗位）

### 4.3 PII 泄露（输出审计）
- ❌ 完整手机 / 邮箱 / 身份证进入 LLM 输出
- ❌ 完整地址 / 家庭成员姓名
- ❌ 候选人照片 / 视频进入 LLM
- ❌ 候选人原始简历文件 URL（含可下载链接）

**审计机制**：所有 LLM 输出经 PII 检测器扫描，命中即阻断 + 警告。

## 5. Human-in-Loop 触发条件（5 类必须审批）

| 触发条件 | 审批人 | SLA |
|---------|--------|-----|
| **发送 offer / 拒绝信** | HR 负责人 | 24h |
| **删除候选人记录** | HR 主管 | 24h |
| **推送薪酬方案至候选人** | 部门 Head + HR | 48h |
| **大批量触达**（>10 人 / 次）| HR 负责人 | 24h |
| **L3 数据访问**（身份证 / 银行 / 医疗）| HR 总监 + CTO | 12h |
| **对外发布候选人信息**（如招聘博客）| PR + HR | 48h |

**实现**：所有触发自动入 `approval` 表 → 通知审批人 → 审批通过后才执行原操作。

## 6. 不确定性显式声明（4 类必须标注）

LLM 必须**显式**标注以下情况，**禁止**幻觉式补全：

1. **数据缺失** — 信息不在 context 中：`# 不知道：[具体字段]`
2. **需人工复核** — 评分置信度 <0.7：`# 需复核：[原因]`
3. **推断 vs 事实** — 非简历原文内容：`# 推断：[内容]`
4. **数据时效** — 信息可能过期（>6 个月）：`# 可能过期：[信息]`

## 7. 异常处理（4 级降级）

| 级别 | 触发 | 处理 |
|------|------|------|
| **L1 软降级** | LLM 临时不可用 | 切换备用模型，重试 1 次 |
| **L2 硬降级** | LLM 持续不可用（>30s）| 返回 `LLM_UNAVAILABLE` 错误 + 建议用户稍后重试 |
| **L3 人工接管** | 关键操作失败 | 通知 HR 接管 + 保留操作日志 |
| **L4 紧急熔断** | 检测到歧视 / 违法 / PII 泄露 | 立即停止 + 告警 + 不返回任何结果 |

## 8. 审计要求

- 所有 Agent 操作记录到 `operation_logs` 表
- 所有写操作（修改候选人 / 发送 offer / 推送）记录到 `audit_logs` 表
- 审计日志保留期：**7 年**（合规要求）
- 审计日志不可修改 / 删除（仅 admin 可读，不可写）
