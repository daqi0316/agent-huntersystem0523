# skill: interview_questions — 面试题库与追问手册

> 本文件是 LLM 工具化技能，按需通过 `load_skill(name="interview_questions")` 加载。
> 内容与 `interview.md` Agent Prompt **不重复**，专注题库结构和追问路径。

## 1. 题目分类体系

| 类别 | 代码 | 描述 |
|------|------|------|
| 行为事件面谈 | BEI | STAR 格式追问过去经历 |
| 技术深度 | TECH | 考察硬技能真实性与深度 |
| 系统设计 | ARCH | 考察架构思维（Senior+）|
| 场景模拟 | CASE | 考察问题解决能力 |
| 文化适配 | CULT | 考察价值观与团队匹配 |
| 动机探测 | MOTI | 考察离职原因/职业规划 |

## 2. BEI 题目模板（按STAR结构）

### 2.1 成就型（正问）
```
请描述一个你最自豪的项目/成果。
- 背景：当时的情况是什么？
- 你的任务：你在团队中承担什么角色？
- 行动：你具体做了哪些事？
- 结果：最终成果如何？量化指标是什么？
```

**追问路径**（按深度递进）：
1. "这个结果在团队/公司是什么水平？"（区分个人 vs 团队）
2. "如果你重新做一次，会有什么不同？"（反思能力）
3. "这个项目的技术难点是什么？你怎么解决的？"（技术真实性）

### 2.2 失败型（逆问）
```
请描述一个你没有达到预期的情况。
- 背景：当时的目标是什么？
- 你的行动：你做了什么？
- 结果：实际结果 vs 预期？
- 反思：你从中学到了什么？
```

**追问路径**：
1. "团队其他人是怎么看待这次失败的？"（归因模式：内归因 vs 外归因）
2. "你现在再做一次，会改进什么？"（成长性）
3. "你后来有没有主动暴露这次失败？"（诚实度）

### 2.3 冲突型（情景）
```
请描述一次与同事/客户/领导的意见分歧，你是怎么处理的？
- 背景：冲突的场景？
- 你的立场 vs 对方立场？
- 你采取了什么行动？
- 结果：关系有没有受影响？
```

## 3. 技术深度题目（按技能领域）

### 3.1 全栈 / 后端
```
BEI: "请描述一个你解决过的最复杂的技术问题。"
追问：
1. "问题的根因是什么？你是怎么定位的？"（排查能力）
2. "你考虑了哪几种方案？为什么选了这个？"（决策能力）
3. "这个问题现在还会发生吗？有没有从根本上解决？"（深度）
4. "如果让你现在重新做，会改进什么？"（反思）

TECH: "请手写一个线程安全的单例模式"（编码题）
TECH: "什么是 CAP 定理？请举例说明"（概念题）
TECH: "如何设计一个分布式 ID 生成器？"（系统设计入门）
```

### 3.2 前端
```
BEI: "请描述一个性能优化的经历。"
追问：
1. "优化前后的性能指标是什么？"
2. "你怎么找到性能瓶颈的？"
3. "除了你说的方案，还有什么其他方案？"

TECH: "什么是虚拟 DOM？它的性能优势在哪里？"（原理）
TECH: "请实现一个 useDebounce Hook"（编码）
TECH: "React 的 reconciliation 算法是怎么工作的？"（深度）
```

### 3.3 数据 / 算法
```
BEI: "请描述一次你从数据中发现业务问题的经历。"
追问：
1. "你是怎么发现这个异常的？"
2. "你提出了什么建议？业务方采纳了吗？"
3. "结果改善了多少？"

TECH: "请解释 L1 / L2 / L3 缓存的区别"（计算机基础）
TECH: "什么是过拟合？怎么避免？"（ML 基础）
```

## 4. 反偏见题目（必问，用于所有候选人）

以下题目用于检测文化适配和偏见，不设标准答案：

| 题目 | 观察点 |
|------|--------|
| "你更喜欢独立工作还是团队协作？为什么？" | 工作风格匹配 |
| "请描述一次你需要说服别人接受你观点的经历" | 沟通方式（攻击性 vs 建设性）|
| "你如何处理同事对你的负面反馈？" | 成长心态 |
| "你见过团队做的最让你不认同的决定是什么？你怎么做的？" | 服从 vs 建设性挑战 |
| "你如何定义成功？" | 价值观（个人成就 vs 团队 vs 客户）|

## 5. 动机探测题目

```
1. "你为什么离开上一家公司？"（真实动机）
   - 理想答案：职业发展 / 寻求挑战 / 价值观不匹配
   - 警戒答案：抱怨老板/同事/公司 / 被动离职未说明

2. "你对我们公司了解多少？"（真实兴趣）
   - 理想答案：了解业务/产品/技术栈，有具体兴趣点
   - 警戒答案：只知道名字或笼统描述

3. "你未来 3 年的职业规划是什么？"（匹配度）
   - 理想答案：与申请岗位发展路径一致
   - 警戒答案：模糊 / 与岗位矛盾 / 只追求晋升

4. "你现在的薪资结构是怎样的？期望是多少？"（理性预期）
   - 理想答案：清晰说明 base/bonus/equity，期望合理
   - 警戒答案：不透明 / 期望远超市场 / 说不清楚
```

## 6. 场景模拟题目（Senior+ 适用）

```
CASE: "你的团队有 3 个人，每个人都说自己很忙，但项目进度已经落后。
  你作为 Tech Lead 会怎么处理？"

考察点：
- 优先级判断（哪些必须做完，哪些可以砍）
- 资源调配（能不能借人/加班/重新分配任务）
- 向上沟通（会不会 escalation 寻求帮助）
- 团队管理（能不能准确评估每个人的真实工作负载）

追问：
1. "你具体怎么评估谁真的很忙？"
2. "如果 deadline 不能改，你会怎么办？"
3. "你会不会告诉老板团队其实完不成？怎么说？"
```

## 7. 题目选择算法

```
def select_questions(candidate_level: str, role_type: str, focus_areas: list[str]) -> list[dict]:
    questions = []
    # 必选：1个 BEI 成就型 + 1个 BEI 失败型 + 1个 MOTI
    questions.append(BEI_ACHIEVEMENT_TEMPLATE)
    questions.append(BEI_FAILURE_TEMPLATE)
    questions.append(MOTIVATION_TEMPLATE)

    # 按级别增加技术深度
    if candidate_level in ("L4", "L5"):
        questions.append(TECH_DEEP_DIVE)
        questions.append(ARCH_SYSTEM_DESIGN)

    # 按岗位类型调整
    if role_type == "frontend":
        questions.append(TECH_FRONTEND)
    elif role_type == "backend":
        questions.append(TECH_BACKEND)
    elif role_type == "fullstack":
        questions.append(TECH_FULLSTACK)

    # 按关注领域补充
    for area in focus_areas:
        if area in AREA_QUESTIONS:
            questions.append(AREA_QUESTIONS[area])

    return questions
```
