# skill: onboarding_workflow — 入职流程与风险信号手册

> 本文件是 LLM 工具化技能，按需通过 `load_skill(name="onboarding_workflow")` 加载。
> 内容与 Agent Prompt **不重复**，专注入职流程节点和风险识别。

## 1. 入职时间线节点

| 阶段 | 时间 | 关键动作 | 负责人 |
|------|------|---------|--------|
| Pre-Day1 | Offer 签署后 | 发送欢迎邮件 + 设备准备 + 工位安排 | HR |
| Day 1 | 入职当天 | 签署合同 + IT 账号 + 工牌 + 团队介绍 | HR + Manager |
| Week 1 | 前 5 个工作日 | 熟悉工作环境 + 核心系统访问 + 1-on-1 了解角色 | Manager |
| Week 2-4 | 第 2-4 周 | 独立完成第一个小任务 + 参加全员会 | Manager |
| Month 2 | 第 2 个月 | 参与项目 + 独立交付能力验证 | Manager |
| Month 3 | 第 3 个月 | 正式评估 + 转正答辩（可选）| Manager + HR |
| Month 6 | 第 6 个月 | 第一次绩效回顾 | Manager |

## 2. Day 1 清单

### 2.1 必完成项（HR）
```
□ 劳动合同签署（原件两份，候选人保留一份）
□ 背调最终确认（如 offer 前未完成）
□ 社保 / 公积金开户（确保当月生效）
□ 个税专项附加扣除确认
□ 员工手册签收
□ 保密协议签署（如适用）
□ IT 账号创建（邮箱 / OA / 代码仓库 / 门禁）
□ 工牌制作
□ 公司资产领用（电脑/手机/门禁卡）
```

### 2.2 必完成项（Manager）
```
□ 发送 Welcome 邮件（包含团队介绍、办公位置、当日安排）
□ 安排座位和设备
□ 指定 Buddy（可选但推荐）
□ 发送 Week 1 Schedule
□ 准备 Day 1 1-on-1（了解候选人的期望和顾虑）
□ 拉入相关群（团队群 / 项目群 / 全员群）
```

## 3. 第一周计划模板

### Day 1 Schedule（示例）
```
09:30 - 10:00  HR 签署合同
10:00 - 11:00  IT 账号激活 + 工具安装
11:00 - 12:00  Manager 欢迎 + 团队介绍
12:00 - 13:30  午餐（团队一起）
13:30 - 14:30  1-on-1 with Manager（了解背景、期望、顾虑）
14:30 - 15:30  参观办公区 + 认识邻居
15:30 - 17:00  阅读团队文档 / 了解项目背景
```

### Week 1 目标
- 熟悉团队成员和协作方式
- 了解当前项目状态和短期目标
- 完成所有系统访问权限申请
- 与直接协作的同事各安排一次 30 分钟 1-on-1

## 4. 风险信号识别

### 4.1 早期危险信号（入职 2 周内）
| 信号 | 可能原因 | 建议动作 |
|------|---------|---------|
| 不主动沟通，总是被动等待任务 | 文化不适应 / 缺乏安全感 | 主动约 1-on-1，了解顾虑 |
| 不参加团队活动/午餐 | 社交障碍 / 不融入 | 安排小组活动，创造交流机会 |
| 系统访问申请拖延 | 流程问题 or 候选人犹豫 | 跟进 IT，确认候选人是否积极 |
| 询问"能不能远程" | 对地点有顾虑 | 确认公司政策，说明实际情况 |
| 反复问同一个流程问题 | 没理解 or 不敢问 | 耐心解答，鼓励提问 |

### 4.2 第一个月危险信号
| 信号 | 可能原因 | 建议动作 |
|------|---------|---------|
| 任务交付延迟，无合理解释 | 能力不足 / 工作量过大 | 1-on-1 了解根因，调整任务 |
| 与同事沟通方式冲突 | 文化不匹配 / 沟通风格差异 | 调解，了解双方视角 |
| 经常私下抱怨团队/公司 | 期望差距 | 了解抱怨点，设定合理预期 |
| 不参加周会/全员会 | 融入问题 / 积极性下降 | 私下沟通，了解原因 |
| 频繁请假 | 个人原因 or 退缩信号 | 关心询问，判断是否正常 |

### 4.3 三个月评估维度
```python
ONBOARDING_ASSESSMENT = {

    }
```

## 5. Manager 1-on-1 话题清单（按阶段）

### Day 1（了解）
- "你之前的工作环境是怎样的？"（对比现在）
- "你对这个岗位最期待的是什么？"
- "你最担心什么？"
- "有什么我们（团队/公司）应该知道的事？"

### Week 1 结束
- "第一周感觉怎么样？"
- "有什么信息你觉得获取不到/很难找到？"
- "团队协作方式上有什么不习惯的？"
- "你的 Buddy 有用吗？还有什么需要我帮忙的？"

### Month 1 结束
- "你觉得自己适应得怎么样？"
- "你觉得自己在哪些地方还需要支持？"
- "第一个月你学到的最重要的东西是什么？"
- "你对团队/流程有什么反馈？"

### Month 3 转正评估
- "你觉得过去 3 个月最有成就感的事是什么？"
- "你觉得最大的挑战是什么？"
- "接下来 3 个月你希望专注什么？"
- "你希望我作为 Manager 做什么不一样的事？"

## 6. 常见入职失败模式

| 模式 | 时间 | 原因 |
|------|------|------|
| "虚假繁荣" | Month 1 | 前几周表现好，之后急剧下降 → 早期给的任务太简单 |
| "孤立无援" | Month 1-2 | 没人带，不融入 → 缺少 Buddy 或 Manager 支持 |
| "期望落差" | Month 2-3 | 入职后发现与面试描述不符 → 面试信息要真实 |
| "能力断层" | Month 2-3 | 基础任务能完成，复杂任务不行 → 面试评估不准确 |
| "文化冲突" | Month 2-3 | 价值观不匹配 → 文化评估要放在面试里 |
