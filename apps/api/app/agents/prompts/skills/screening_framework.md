# skill: screening_framework — 筛选决策框架参考手册

> 本文件是 LLM 工具化技能，按需通过 `load_skill(name="screening_framework")` 加载。
> 内容与 `screening.md` 不重复，专注决策框架和校准方法论。

## 1. 四维评分权重（默认配置）

| 维度 | 权重 | 说明 |
|------|------|------|
| 硬性条件匹配（Hard Skills）| 40% | 技能/经验/学历/行业必须满足 |
| 软性素质（Soft Skills）| 30% | 沟通/学习/文化匹配 |
| 动机与稳定性（Motivation）| 20% | 求职动机/稳定性/期望匹配 |
| 潜力与成长性（Growth）| 10% | 成长轨迹/影响力/创新 |

**特殊情况权重调整**：
- 社招高管（年薪 > 80W）：硬性 30% / 软性 30% / 动机 20% / 潜力 20%
- 应届生：硬性 30% / 软性 30% / 动机 20% / 潜力 20%
- 实习岗：硬性 20% / 软性 40% / 动机 25% / 潜力 15%

## 2. 硬性条件判断树

```
JD 必选项（Must-have）:
  ├── 技能匹配度 < 60% → 直接淘汰（STOP）
  ├── 工作经验年限不足 → 直接淘汰（STOP）
  ├── 学历不满足 → 直接淘汰（STOP）
  └── 行业背景要求未达 → 降级处理

JD 加分项（Nice-to-have）:
  └── 每满足一项 +0.5 分（满分 3 分）
```

**技能匹配度计算**：
```
match_score = (JD必备技能中候选人所占比例) × 0.6
            + (JD加分技能中候选人所占比例) × 0.3
            + (额外高相关技能) × 0.1
```

## 3. 软性素质评估细则

### 3.1 沟通能力（评估来源：项目描述措辞）
| 证据 | 得分 |
|------|------|
| 能清晰描述问题→方案→结果（STAR 格式）| 9-10 |
| 描述有逻辑，但结果量化不足 | 7-8 |
| 描述笼统，缺乏具体细节 | 5-6 |
| 描述混乱，逻辑不清 | 1-4 |

### 3.2 学习能力
| 证据 | 得分 |
|------|------|
| 主动学习新技能且有量化成果（证书/项目）| 9-10 |
| 技术栈持续更新，有跟进意识 | 7-8 |
| 技术栈单一，未见主动拓展 | 3-6 |
| 技术栈过时且无更新意愿 | 1-2 |

### 3.3 文化匹配
评估维度：
- 工作风格（独立 vs 协作）
- 规模偏好（大厂 vs 创业）
- 行业偏好（稳定 vs 快速变化）
- 汇报偏好（扁平 vs 层级）

## 4. 动机与稳定性评估

### 4.1 跳槽频率分析
| 模式 | 信号 | 建议 |
|------|------|------|
| 每份工作 > 3 年 | 稳定型 | 正常评分 |
| 2-3 年换一次 | 正常 | 正常评分 |
| 1-2 年换一次 | 偏频繁 | 结合原因判断 |
| < 1 年换一次（3次以上）| 高风险 | 严格审查 |
| 空档期 > 6 个月 | 需解释 | 了解原因 |

### 4.2 求职动机判断
强动机信号：
- 主动投递（而非海投）
- 对公司业务有具体了解（面试中体现）
- 期望薪资与市场接近
- 通勤距离合理

弱动机信号：
- 大量海投（多处同时面试）
- 对公司几乎不了解
- 期望薪资远高于市场
- 通勤明显不便

## 5. 潜力评估（对于 Junior 候选人权重提升）

### 5.1 成长轨迹评分
| 证据 | 得分 |
|------|------|
| 持续晋升（每 2 年一次）| 9-10 |
| 稳定晋升（3-4 年一次）| 7-8 |
| 平级调动但职责扩大 | 5-6 |
| 长期同一级别 | 1-4 |

### 5.2 影响力证据
优先证据（按可靠性排序）：
1. 专利 / 顶会论文 / 开源主导项目
2. 技术博客 / 演讲 / 社区贡献
3. 团队规模（主导 / 参与）
4. 业务指标量化（收入/用户量/效率提升）
5. 流程改进 / 工具建设

## 6. 校准规则（Reviewer 一致性）

为减少评审者主观差异，强制执行以下规则：

**校准会议**（每季度一次）：
- 随机抽取 10 份已筛选简历集体重评
- 讨论分歧 > 2 分的案例
- 更新决策边界案例库

**决策边界案例库**：
| 场景 | 决策规则 |
|------|---------|
| 经验 5 年，学历大专 | 除非有突出项目成果或稀缺技能，否则降级 |
| 薪资期望比市场高 50% | 需候选人证明市场低估（offer 对比） |
| 跳槽频率高但每次有晋升 | 正常评分（晋升可抵消频繁跳槽） |
| JD 写了 10 项要求 | 必须满足至少 7 项（70%）|

## 7. 综合决策规则

```
if overall_score >= 8.0:
    recommendation = "strong_recommend"
elif overall_score >= 7.0:
    recommendation = "recommend"
elif overall_score >= 5.5:
    recommendation = "consider"
elif hard_skills_score < 6.0:
    recommendation = "reject"  # 硬性条件不满足，一票否决
else:
    recommendation = "consider"  # 有软性潜力，但硬性一般
```

**一票否决条件**（直接 reject，不看总分）：
1. 硬性条件匹配度 < 50%
2. 最近一份工作在职时间 < 3 个月且无合理解释
3. 学历造假证据（与其他来源矛盾）
4. 简历中有明显虚假项目经历（无法通过背调验证）
