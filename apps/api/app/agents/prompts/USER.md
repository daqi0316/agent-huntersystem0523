# USER.md — HR 用户画像默认模板（per user 副本存储）

> 本文件是**默认模板**。每个登录的 HR 用户首次访问时，系统会从本模板复制一份到 `settings/users/{user_id}/memory.md`，用户可独立编辑。
> 
> 运行时文件位置：`apps/api/runtime/users/{user_id}/memory.md`（.gitignore 排除）
> API：`GET/PUT /api/v1/users/me/memory`（仅本人可读写，admin 只读）
> 写操作记录到 `audit_logs` 表。

## 1. 基本信息

```yaml
user_id: <从 auth 获取>
display_name: <HR 真实姓名>
role: <HR Manager | Recruiter | HR Director | Hiring Manager>
years_of_experience: <3-5 | 5-10 | 10+>
focus_areas: [<技术招聘 | 校招 | 高端猎聘 | 全模块>]
tenant: <公司 / 子公司名称>
```

## 2. 输出偏好

| 维度 | 默认 | 备选 |
|------|------|------|
| **输出格式** | Markdown + 表格 | JSON / 纯文本 / 流程图 |
| **量化要求** | 所有建议附 KPI / 数据 | 定性即可 |
| **决策风格** | 结果导向 + 体系化 | 探索性 / 对比分析 |
| **结构化程度** | 高（bullet / 表格 / 标题）| 中（段落）|
| **沟通语言** | 中文 | 英文 / 中英混合 |
| **详细度** | 适中（不省略关键步骤）| 简洁（结论先行）|

## 3. 关注指标（4 大 KPI）

按重要度排序，**每次主动汇报**：

1. **试用期通过率** — 90 天 / 180 天转正比例，target ≥ 90%
2. **关键岗位留存率** — 1 年留存 ≥ 85%、3 年留存 ≥ 65%
3. **招聘漏斗转化率** — 申请→offer 转化率，target ≥ 5%
4. **渠道 ROI** — 单位招聘成本的目标达成数，target < ¥8K / P6

**次要指标**（按需提供）：
- 招聘周期（time-to-fill）< 35 天
- Offer 接受率 ≥ 80%
- 候选人 NPS ≥ 50
- HR 服务满意度 ≥ 4.5/5

## 4. 学习模式

| 维度 | 默认 | 备选 |
|------|------|------|
| **学习方式** | 项目驱动（通过复刻案例学习）| 理论先行 / 课程驱动 |
| **基础覆盖** | 渐进式（不省略基础步骤）| 跳跃式（高阶优先）|
| **互动频率** | 高频确认（每步给进度）| 一次性输出 |
| **错误处理** | 显式标注（错哪 + 为什么 + 怎么改）| 静默重试 |

## 6. 不喜欢 / 屏蔽项

```yaml
# 可选：明确告知 Agent 不要做的事
avoid:
  - 重复"换行不报错"风格的废话
  - 长段落无 bullet 的输出
  - 不附依据的结论

```

## 7. 协作风格

| 维度 | 偏好 |
|------|------|
| **主动程度** | 主动给建议（不等问） |
| **冲突处理** | 直接指出 + 备选方案（不绕弯） |
| **确认机制** | 关键决策前确认（避免方向错） |
| **复盘频率** | 每月 1 次（提取成功模式） |
